# now/ — routing catalog

Fleeting working state — the **Tier-1 surface**, read every session. UPDATE-IN-PLACE; history lives
in the version-control log. This is the *first* thing an agent reads after `CONVENTIONS.md`. Route
through here; don't browse. Schema authority: `CONVENTIONS.md`.

<!-- On install, the concierge instantiates each `*.template.md` here by filling the placeholder tokens
     and dropping the `.template` suffix, so the live files below exist and this index resolves. -->

## Read every session (in this order)

- ⭐ `status.md` — **Open when:** session start, or "where are we?" **Carry-away:** the current
  headline state — active work-unit, branch, what builds/runs, what changed since last session.
- ⭐ `work-plan.md` — **Open when:** deciding what to do next, or checking a locked decision.
  **Carry-away:** the plan (phases / milestones), the immediate next action, and the session's locked
  decisions.
- ⭐ `open-questions.md` — **Open when:** something is undecided or blocking; before assuming a
  default. **Carry-away:** `OQ-NNN` is the single source (inline-RESOLVED threading); a resolved OQ
  cites the ADR/WU that closed it.

- ⭐ `obligations.md` *(present on multi-party installs; single-party carries the same content as an
  `## Obligations` section inside `handoff.md`)* — **Open when:** session start, and before assuming a
  counterparty (another agent, another repo, the operator) will deliver or acting on a silent wait.
  **Carry-away:** inter-party debts both directions + the pre-decided **default-if-silent** at each
  receivable's trigger; a row POINTS at an `OQ-`/`WU-`/`REV-`/DEFER id, never duplicates it. Schema:
  ADR-0012.

## Generated at checkpoints (present after the first `/handoff`)

- `handoff.md` — **Open when:** resuming after time away (consumed by `/orient`). **Carry-away:** the
  curated cold-start brief + anti-assumptions + the detour-chain; if `last-modified > 24h` or it
  contradicts the repo, re-`/orient`; if badly stale, `/handoff` first. Regenerated by `/handoff`,
  verified by `/orient`.

## Nested — the lessons Tier-1 surface (`now/lessons/`)

- `lessons/MOC.md` — **Open when:** auto-loaded at session start; the bounded (~30 entries) map-of-
  content over the `lessons/` ledger. **Carry-away:** evergreen lessons read before action; the full
  ledger lives in `lessons/` (route via `lessons/index.md`).
- `lessons/proposals.md` — **Open when:** running the lesson-distillation pass or the `/handoff`
  promotion gate. **Carry-away:** candidate lessons staged for human-gated promotion; "0 high-quality
  proposals" beats "3 weak ones".

## Root journal

- `../log.md` — **Open when:** appending a session/operation entry, or scanning recent history.
  **Carry-away:** the ONE canonical append-only journal (newest-at-top), lives at the `.agent-docs/`
  root, not here.

## Maintenance

UPDATE-IN-PLACE as a byproduct of work — `/flush` mid-session, `/handoff` at checkpoints.
`last-modified` staleness is lint-checked. The `now/lessons/` MOC + proposals are the human-gated
lessons surface.
